## Getting started

The package is hosted on **GitHub Packages**, not public npm. Add a scoped `.npmrc` first:

```bash
# .npmrc (in your consuming project root)
@studio-manfred:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
```

Then install and import:

```bash
npm install @studio-manfred/manfred-design-system
```

```tsx
import '@studio-manfred/manfred-design-system/styles';
import { Button } from '@studio-manfred/manfred-design-system';

export function App() {
  return <Button variant="brand">Hello, Manfred</Button>;
}
```

Full onboarding (PAT setup, CI examples, troubleshooting): see the [Consuming guide](https://github.com/Studio-Manfred/manfred-design-system/blob/main/docs/CONSUMING.md).

## Light, dark, and system

Use the **theme switcher** in the Storybook toolbar (top right) to flip between `system` (follows OS), `light`, and `dark`. Every story is verified clean by the runtime axe scan in both modes — color contrast, focus indicators, and label semantics are part of the contract.

---

## Working with the system

- **Tokens are the source of truth.** Components reference `var(--color-...)` directly. Don't hardcode hex.
- **Composition over configuration.** `Card` is `Card + CardHeader + CardContent + CardFooter` — pick what you need.
- **Layout primitives, then components.** Reach for `Stack` / `Grid` / `Container` before raw Tailwind utilities.
- **Accessibility is non-negotiable.** Every interactive component supports keyboard, screen reader, and reduced-motion. The runtime axe scan blocks anything that regresses.
